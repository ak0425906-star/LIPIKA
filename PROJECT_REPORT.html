Comprehensive Project Principles & Methodologies

1. Core AI Principle: Siamese Neural Networks

LIPIKA does not use standard classification. Instead, it employs One-Shot Learning using Siamese Networks. This allows the system to verify a student's handwriting even if it has only seen 2-3 previous samples.

Sub-network Architecture

Two identical CNNs (Convolutional Neural Networks) process the reference and the assignment images simultaneously, sharing the exact same weights and parameters.

Feature Embedding

Images are compressed into a 128-dimensional vector. This vector represents the "DNA" of the handwriting, capturing stroke tilt, pressure patterns, and loop sizes.

2. The Dynamic Calibration Algorithm

By raising the normalized score to the 10th power, we create a logarithmic-style penalty for variance. Even a 5% difference in the raw ML features results in a 40% drop in the final score. This ensures that only Strong Matches (≥85%) are considered authenticated.

3. Backend Engineering & Data Integrity

A. Asynchronous Request Handling

Using FastAPI's async/await syntax, the server implements non-blocking I/O operations. While the ML model processes a comparison (CPU intensive), the main thread remains free to handle other student submissions.

B. Stateless Authentication (JWT)

Authentication is handled via JSON Web Tokens. This follows the REST principle of "Statelessness," where the server does not store session data. Instead, the identity and role (Student/Teacher/Admin) are securely encoded and signed within the token itself.

C. Relational Database Normalization

The SQLite database is structured in 3rd Normal Form (3NF) to eliminate data redundancy. Foreign Key constraints ensure that assignments are linked to existing tasks, and tasks are linked to valid subjects.

4. Exhaustive Viva Preparation

Question: Why use a Siamese Network instead of a standard CNN? Answer: Standard CNNs require thousands of images per person to "learn" a face or handwriting. Siamese Networks only need a few samples (One-Shot Learning) because they learn to distinguish *differences* rather than specific *identities*.
Question: What is the significance of the "Minimum Score" selection? Answer: A student might have varied handwriting across different days. By comparing the assignment against *all* references and taking the minimum similarity, we ensure the system is at its most skeptical level, prioritizing security over convenience.
Question: How does the system prevent Directory Traversal attacks? Answer: We use the uuid library to rename every uploaded file into a random 36-character string. This prevents attackers from guessing file paths or uploading files that could overwrite system configurations.
Question: Explain the "Event Loop" in your backend. Answer: FastAPI runs on an Event Loop. When an ML request is made, the task is sent to the background, and the loop continues to listen for new requests. When the ML result is ready, a callback triggers the final response. This allows for high concurrency.
